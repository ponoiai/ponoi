# Ponoi — план визуального паритета «1-в-1» с прототипом

Цель: довести настоящий app (`ponoiai/ponoi`, Vite+React+TS+Supabase) до внешнего вида и UX прототипа
(`Ponoi_patched.html`). Прототип — скомпилированный бандл без исходника, поэтому дизайн
воссоздаётся по извлечённым CSS-токенам и структуре, а не копируется покомпонентно.
Каждый проход коммитится отдельно; проверка — пользователем локально (`npm run dev`), т.к.
агент не может собрать/запустить (нет Node/npm, SCAS недоступен).

## Инвентарь семейств компонентов прототипа
- `pql-*` — общие элементы (кнопки, модалки, тосты, attach-карточки).
- `pqs-*` / `pqp-*` — экран «Настройки пользователя»: вкладки Мой аккаунт, Профиль,
  Конфиденциальность, Голос и видео, Профиль лица, Внешний вид, Уведомления,
  Горячие клавиши, Язык, Дисплей, Активность, Дополнительно.
- `pfp-*` — полный профиль пользователя; `pmini-*` — мини-профиль (popout по клику на аватар).
- `pfr-*` — Друзья (вкладки В сети / Все / Заявки, добавить по коду).
- `pcl-*` / `vconn-*` / `camera-*` — звонок: сетка участников, микрофон/камера/экран, статус связи.
- `pmf-*` / `pmus-*` — Ponoi Music (полноэкранный плеер + мини-плеер).
- `phv-*` — hover-тулбар над сообщением; контекст-меню сообщения (`__ponoiMsgMenu`).

## Проходы (по порядку)

### Проход 1 — Каркас чата и панелей «как в Discord» ✅ (commit b14ad76)
- Плотный (компактный) режим сообщений: группировка подряд идущих сообщений одного автора,
  дата-разделители, hover-подсветка строки, hover-тулбар (реакция/ответ/ещё).
- Сайдбар серверов: пилюля-индикатор активного/непрочитанного слева, тултипы по наведению.
- Сайдбар каналов: секции «ТЕКСТОВЫЕ / ГОЛОСОВЫЕ», иконки #/🔊, шапка сервера с ▾.
- Шапка чата: иконка канала, название, разделитель, справа — поиск/закреплённые/участники/звонок.

### Проход 2 — Панель участников + статусы + профили ✅ (commit 6302392)
- Правый сайдбар «Участники»: группировка по ролям (Администратор/Модератор/Участник),
  цветные ники по роли, аватар со статус-точкой.
- Статусы присутствия: В сети / Нет на месте / Не беспокоить / Не в сети (цветная точка на аватаре).
- Мини-профиль (`pmini-*`) — popout по клику на аватар/ник: шапка-баннер, аватар, ник,
  роли-теги, «О себе», кнопки Написать/Позвонить.
- Полный профиль (`pfp-*`): изменить аватар/шапку/тему, «О себе».

### Проход 3 — Экран «Настройки пользователя» (`pqs-*`) ✅
- Полноэкранный layout: слева список категорий, справа контент, крестик «Закрыть (Esc)».
- Вкладки: Мой аккаунт, Профиль, Внешний вид (тема/акцент/компактный режим/размер шрифта/анимации),
  Уведомления (системные/звуки/упоминания/счётчик), Голос и видео (устройства, громкости,
  проверка микрофона, чувствительность), Горячие клавиши, Язык (карточки-флаги),
  Дисплей (зум), Конфиденциальность (ЛС/сбор данных), Активность, Дополнительно (devmode/очистка).
- Персист: Supabase `profiles` (аккаунт/о себе/тема) + localStorage для клиентских (зум/компактность/язык).

### Проход 4 — Ponoi Music «как в прототипе»
- Полноэкранный плеер `pmf-*`: обложка, крутящаяся пластинка, seek-бар, громкость,
  очередь/плейлисты, добавление трека (файл/URL/онлайн), кнопки-карточки при hover.
- Настройки плеера ⚙: гифки по бокам + фон (фото/видео ≤15с, IndexedDB), затемнение (уже есть — оформить).
- Мини-плеер `pmus-*` в углу.

### Проход 5 — Звонки/видео/экран `pcl-*`
- Экран звонка: сетка участников (`pcl-ball`/видеотайлы), статус связи, «Звоним…»,
  панель: Микрофон/Камера/Демонстрация экрана/Отключиться, индикаторы дБ.

### Проход 6 — Прототип-фичи поверх дизайна
- Кастомные эмодзи, продвинутый GIF-пикер, реакции, закреплённые сообщения,
  контекст-меню сообщения (Ответить/Переслать/Копировать/Закрепить/Добавить в мои фото),
  питомец, темы оформления, тосты, Ctrl+V вставка.

## Принципы
- Имена классов настоящего app сохраняются; дизайн наращивается, компоненты не ломаются.
- Каждый проход = отдельный commit + краткий чейнджлог тут (обновлять статус галочками).
- После каждого прохода пользователь делает git pull / npm run dev и даёт предметный фидбэк;
  дизайн добивается итеративно под конкретные замечания.
