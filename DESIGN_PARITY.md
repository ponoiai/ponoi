# Ponoi — план визуального паритета «1-в-1» с прототипом

Цель: довести настоящий app (`ponoiai/ponoi`, Vite+React+TS+Supabase) до внешнего вида и UX прототипа
(`Ponoi_patched.html`). Прототип — скомпилированный бандл без исходника, поэтому дизайн
воссоздаётся по извлечённым CSS-токенам и структуре, а не копируется покомпонентно.
Каждый проход коммитится отдельно; проверка — пользователем локально (`npm run dev`), т.к.
агент не может собрать/запустить (нет Node/npm, SCAS недоступен).

## Инвентарь семейств компонентов прототипа
- `pql-*` — общие элементы (кнопки, модалки, тосты, attach-карточки).
- `pqs-*` / `pqp-*` — экран «Настройки пользователя»: вкладки Мой аккаунт, Профиль,
  Конфиденциальность, Голос и видео, Профиль лица, Внешний вид, Уведомления,
  Горячие клавиши, Язык, Дисплей, Активность, Дополнительно.
- `pfp-*` — полный профиль пользователя; `pmini-*` — мини-профиль (popout по клику на аватар).
- `pfr-*` — Друзья (вкладки В сети / Все / Заявки, добавить по коду).
- `pcl-*` / `vconn-*` / `camera-*` — звонок: сетка участников, микрофон/камера/экран, статус связи.
- `pmf-*` / `pmus-*` — Ponoi Music (полноэкранный плеер + мини-плеер).
- `phv-*` — hover-тулбар над сообщением; контекст-меню сообщения (`__ponoiMsgMenu`).

## Принципы
- Имена классов настоящего app сохраняются; дизайн наращивается, компоненты не ломаются.
- Каждый проход = отдельный commit + краткий чейнджлог тут (обновлять статус галочками).
- После каждого прохода пользователь делает git pull / npm run dev и даёт предметный фидбэк;
  дизайн добивается итеративно под конкретные замечания.

### v1.40.0 — Ник и юзернейм доделаны + настройки с подтверждением ✅
- «Друзья → Добавить в друзья»: в «Мой юзернейм» теперь показывается настоящий юзернейм (раньше — ник).
- Поиск людей ищет и по нику, и по юзернейму; в результатах ник крупно + юзернейм мелко.
- Мини-профиль: под ником — настоящий юзернейм из базы (раньше — ник в нижнем регистре); «Скопировать юзернейм» копирует юзернейм.
- Заявки в друзья подписываются ником (display_name), а не юзернеймом.
- «Мой аккаунт»: в карточке — ник крупно, юзернейм мелко под ним (как в Discord).
- Цвета темы профиля больше не сохраняются мгновенно — только через плашку «Есть несохранённые изменения!» (Сбросить / Сохранить изменения), как всё остальное.
- После сохранения ник обновляется сразу во всём приложении (событие ponoi-profile-updated), без перезагрузки.

### v1.42.0 — Надёжность: настройки и отправка сообщений ✅
- Настройки нельзя закрыть (Esc/крестик) с несохранёнными изменениями — плашка трясётся, как в Discord;
  закрыть можно только после «Сбросить» или «Сохранить изменения».

- После сохранения поля показывают ровно то, что сохранилось (ник/юзернейм подставляются обратно).
- Загрузка профиля в настройках: трёхступенчатый фолбэк по миграциям (username_changed_at → display_name → username);

- Композер: синхронный замок от двойной отправки (два быстрых Enter больше не шлют сообщение дважды);
