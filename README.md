# Ponoi (настоящее приложение)

Это исходник Ponoi как **настоящего** приложения: Vite + React + TypeScript + **Supabase**
(регистрация/вход, сервера, каналы и сообщения в реальном времени между устройствами и пользователями).
Он заменяет прежние браузерные live-патчи на реальное приложение с бэкендом.

## Что уже есть (этап 1 — фундамент)
- Регистрация и вход по email/паролю (Supabase Auth).
- Профили пользователей.
- Сервера и каналы (создание).
- Сообщения в реальном времени (Supabase Realtime) — видны на всех устройствах и у всех пользователей.
- Discord-подобный тёмный UI.

## Что дальше (следующие этапы)
- Друзья и личные сообщения (DM).
- Участники серверов + приглашения (таблица `server_members`, ужесточение RLS).
- Загрузка файлов/аватаров (Supabase Storage).
- Голосовые/видео-звонки и демонстрация экрана (WebRTC / LiveKit).
- Перенос «фишек» из прошлого прототипа: Ponoi Music, гифки, кастомные эмодзи, питомец и т.д.

## Установка
1. Установи Node.js 18+.
2. В папке проекта:
   ```bash
   npm install
   ```
3. Создай проект на https://supabase.com (бесплатно).
4. Supabase Dashboard -> SQL Editor -> вставь и выполни `supabase/schema.sql`.
5. Supabase Dashboard -> Project Settings -> API -> скопируй **Project URL** и **anon public key**.
6. Скопируй `.env.example` в `.env` и подставь значения:
   ```
   VITE_SUPABASE_URL=...
   VITE_SUPABASE_ANON_KEY=...
   ```
7. Запусти:
   ```bash
   npm run dev
   ```
8. Открой ссылку из терминала, зарегистрируйся, создай сервер (＋) и пиши сообщения.
   Открой второе окно/устройство, войди другим аккаунтом — сообщения приходят в реальном времени.

## Деплой
- `npm run build` -> статика в `dist/` (можно разместить на Vercel/Netlify/Cloudflare Pages).
- Supabase — уже облачный бэкенд, отдельный сервер не нужен.
